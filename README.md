* 네트워크, 지도, 관계 표현에 활용

| 방법     | 특징              |

## ✅ 문제 1. 인접 리스트 기반 그래프 ADT 구현

### 📌 문제 개요

무방향 그래프를 인접 리스트로 표현하고,
정점과 간선을 추가 및 출력하는 그래프 ADT를 구현한다.

### 📂 관련 파일

* `graph_adjlist.c`

### 🔹 구현 내용

* 그래프 초기화
* 간선 추가 (무방향)
* 인접 리스트 출력

### 🔹 핵심 포인트

* 연결 리스트를 이용한 인접 정점 관리
* 무방향 그래프이므로 간선은 양방향으로 저장

---

## ✅ 문제 2. 스택을 이용한 DFS 구현 (비재귀)

### 📌 문제 개요

재귀를 사용하지 않고 **스택(Stack)** 을 이용하여
그래프의 DFS(깊이 우선 탐색)를 구현한다.

### 📂 관련 파일

* `dfs_stack.c`

### 🔹 구현 내용

* 사용자 정의 스택 구현
* 방문 여부를 확인하는 `visited[]` 배열 사용
* 시작 정점부터 깊이 우선 탐색 수행

### 🔹 핵심 포인트

* DFS는 본질적으로 스택 구조를 사용함
* 재귀 DFS는 시스템 스택을 사용
* 스택 DFS는 사용자 정의 스택을 사용

### 🔹 DFS 특징

* 한 방향으로 최대한 깊이 탐색 후 되돌아옴
* 방문 순서는 인접 정점 push 순서에 따라 달라질 수 있음

---

## ✅ 문제 3. 최소 비용 신장 트리 (Kruskal 알고리즘)

### 📌 문제 개요

가중치가 있는 무방향 그래프에서
**최소 비용 신장 트리(MST)** 를 Kruskal 알고리즘으로 구현한다.

### 📂 관련 파일

* `kruskal_mst.c`

### 🔹 구현 내용

* 간선 정보를 구조체로 관리
* 간선을 가중치 기준으로 정렬
* Union-Find 자료구조로 사이클 방지
* MST의 총 비용 계산

### 🔹 핵심 포인트

* MST는 모든 정점을 포함하면서 사이클이 없는 트리
* 간선 개수는 항상 `정점 수 - 1`
* Kruskal은 Greedy 알고리즘

---

## 🔧 Union-Find 자료구조 요약

### ▪ Find

* 특정 원소가 속한 집합의 대표를 찾는 연산

### ▪ Union

* 두 집합을 하나로 합치는 연산

### ▪ 역할

* Kruskal 알고리즘에서 사이클 발생 여부 판단

## 🧠 시험 대비 핵심 정리

* DFS는 스택 기반 탐색이다.
* 재귀 DFS = 시스템 스택 사용
* MST는 모든 정점을 포함하며 사이클이 없다.
* Kruskal 알고리즘은 간선을 정렬한 뒤 순차적으로 선택한다.
* Union-Find는 사이클 판별에 사용된다.
